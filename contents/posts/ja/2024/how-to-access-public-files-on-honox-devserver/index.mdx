x-basicのテンプレートで作成したHonoXプロジェクトのルートディレクトリにある`public`の配下のファイルは、開発サーバーではアクセスができなかった。

[@hono/vite-dev-server](https://github.com/honojs/vite-plugins/tree/main/packages/dev-server#exclude)のREADMEに、`public/assets/*`配下の静的ファイルを返したければ`exclude`で`/assets/*`を除外することと書かれていた。

例えば、`public`配下にある拡張子がwebp, png, svgの画像を開発サーバーでアクセスするには、次のようにexcludeの設定することで対応できる。

```javascript
export default defineConfig(({ mode }) => {
  return {
    plugins: [
      :
      honox({
        devServer: {
          adapter,
          exclude: [
            ...devServerDefaultOptions.exclude,
            /\.(webp|png|svg)$/,
          ],
        },
      }),
      :
```
